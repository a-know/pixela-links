# Phase 0-5：画面設計

## 画面一覧と遷移

```
┌─────────────────────────────┐
│  アカウント設定画面（初回のみ）  │
│  username / token 入力       │
│  → 接続確認                  │
└─────────────┬───────────────┘
              ↓ 設定完了
┌─────────────────────────────┐
│        ホーム画面             │
│  データ種別リスト（32種）      │
│  各行に：名前・状態・エラー数   │
└──┬──────────────────────────┘
   ↓ 行タップ
┌─────────────────────────────┐
│      データ種別 詳細・設定画面  │
│  有効/無効トグル               │
│  グラフID入力                 │
│  最終送信日時                 │
│  エラー履歴                   │
└─────────────────────────────┘
```

## 各画面の詳細

### 1. アカウント設定画面

初回起動時または未設定時に表示。

```
┌─────────────────────────────┐
│  Pixelaアカウント設定          │
│                             │
│  ユーザー名  [a-know        ] │
│  APIトークン [***********   ] │
│                             │
│       [接続を確認する]         │
│  ✓ 接続できました             │
│       [保存する]              │
└─────────────────────────────┘
```

---

### 2. ホーム画面

データ種別をカテゴリごとにセクション分けして表示。

```
┌─────────────────────────────┐
│  ⚙ a-know                   │  ← アカウント設定へ
├─────────────────────────────┤
│  HealthKit                  │
│  ✓ 歩数          今日 0エラー│
│  ✓ 歩行距離       今日 2エラー│  ← エラーは赤表示
│    走行距離       （未設定）  │
│    ...                      │
├─────────────────────────────┤
│  写真・メディア               │
│    カメラロール追加枚数（未設定）│
│    ...                      │
├─────────────────────────────┤
│  ...                        │
└─────────────────────────────┘
```

**表示ルール**
- 有効・設定済み：チェックマーク + 今日のエラー件数
- 未設定：グレー表示
- ハードウェア非対応：グレーアウト + 理由（例：「Apple Watch が必要です」）
- `backgroundReliability` が `.low` の項目：「⚠ 精度が下がる場合があります」注記

## 補足事項

- 設定済みでも `isAvailableOnDevice` が false の場合：設定画面は表示するが送信は実行せず「このデバイスでは利用できません」を表示
- HealthKit 権限が拒否されている場合：設定画面から iOS の設定アプリへ誘導
- エラー履歴は直近30件程度を表示（古いレコードは定期パージ済みのため）
